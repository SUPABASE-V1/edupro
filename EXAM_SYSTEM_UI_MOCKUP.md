## ?? Color Scheme & Styling

**All components use `design.css` classes:**

### Primary Colors
- Primary: `var(--primary)` - Blue (#4B9CEF)
- Success: `var(--success)` - Green (#34C759)
- Warning: `var(--warning)` - Orange (#FF9500)
- Danger: `var(--danger)` - Red (#FF3B30)

### Component Classes
- Cards: `.card` - White background, border, shadow
- Buttons: `.btn`, `.btnPrimary`, `.btnSecondary`
- Inputs: `.input` - Consistent styling
- Sections: `.section`, `.sectionTitle`
- Icons: `.icon16`, `.icon20` - Sized consistently

### Visual Effects
- **Hover**: Cards lift with shadow, buttons darken
- **Active**: Primary color border/background
- **Focus**: Blue outline for accessibility
- **Transitions**: All 0.2s ease for smoothness

### Spacing
- Consistent use of `var(--space-X)` (1-6)
- Cards: `var(--space-3)` padding
- Sections: `var(--space-4)` margin-bottom
- Grid gaps: `var(--space-3)`

### Typography
- Headers: Bold, larger sizes
- Body: 14-15px, readable line-height
- Muted text: `color: var(--muted)` for secondary info
- Icons next to text for visual hierarchy

## ? Key UX Features

1. **Empty States**: All components show helpful messages when empty
2. **Loading States**: Spinner/text while data loads
3. **Hover Effects**: Visual feedback on interactive elements
4. **Mobile-First**: Responsive grid/flex layouts
5. **Accessibility**: Proper contrast, focus states, semantic HTML
6. **Consistency**: Same design patterns across all components
7. **Performance**: Optimized images, lazy loading where appropriate
